## 需求摘要

详见 e000002 `requirement_defined` payload，核心要点：

### 业务模型
- 用户：admin/user 角色，管理员管理用户
- 合同：CRUD + 状态流转（draft→pending_review→active→expired，可终止）
- 附件：pdf/doc/docx，单文件 ≤10MB
- 审计日志：关键操作记录

### 产品流程
1. 登录页 → 用户名+密码认证
2. 用户管理（管理员）→ 用户列表、创建、启用/禁用
3. 合同列表 → 搜索、状态筛选、分页
4. 合同创建 → 填写信息，保存至 draft
5. 合同详情 → 查看、编辑、附件上传/下载/删除、状态流转
6. 异常路径 → 未登录重定向、无权限 403、附件校验

### 标准交付基线
- 后端：Python FastAPI + SQLite
- 前端：Jinja2 模板 + 原生 CSS/JS
- 数据库：schema + 初始化数据（演示账号）+ 迁移
- 自测：pytest（登录/用户管理/合同CRUD/附件校验）
- 文档：architecture.md、runbook.md、decisions/、iterations/

### 数据约束
- 演示账号：admin/admin123、user/user123
- 全部假数据，不得输入真实信息

### 明确不做
- OAuth/OIDC、电子签名、多租户、邮件通知、移动端、批量导入导出
